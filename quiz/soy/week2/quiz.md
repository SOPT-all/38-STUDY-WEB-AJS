## Q1. 빈칸 채우기

```text
실행 컨텍스트는 소스코드를 실행하는 데 필요한 환경을 제공하고, 코드의 실행 결과를 관리하는 영역이다. 실행 컨텍스트에서 식별자와 스코프는 ( A )로 관리하고, 코드 실행 순서는 ( B )로 관리한다.

( A )는 식별자와 식별자에 바인딩된 값을 관리하는 ( C )와, 상위 스코프를 참조하는 ( D )로 구성된다.
이때 ( D )는 상위 코드의 렉시컬 환경을 가리키며, 이를 통해 스코프가 단방향 링크드 리스트 형태로 연결된 ( E )가 구현된다.

ES6 이후 전역 환경 레코드는 전역 선언의 종류에 따라 두 영역으로 나뉜다.
var 키워드로 선언한 전역 변수와 함수 선언문은 ( F )에서 관리되고, let, const 키워드로 선언한 전역 변수는 ( G )에서 관리된다.

( F )는 전역 객체인 ( H )와 연결되어 있기 때문에, var로 선언한 전역 변수는 전역 객체의 프로퍼티가 된다. 따라서 window.x처럼 전역 객체를 명시하지 않아도 x로 참조할 수 있다.

반면 let, const로 선언한 전역 변수는 전역 객체의 프로퍼티가 되지 않고 ( G )에 존재한다. 또한 변수 선언문에 도달하기 전까지는 ( I )에 빠져 참조할 수 없다.

자바스크립트 엔진은 소스코드를 실행할 때 먼저 ( J ) 과정을 거쳐 선언문을 실행하고 식별자를 ( C )에 등록한다. 이후 ( K ) 과정에서 선언문을 제외한 코드를 순차적으로 실행한다.
```

## Q3. 아래 코드의 실행 결과(에러 발생 시 에러 종류 포함)를 쓰고, `bar` 함수 실행 시 `x`, `y`, `z` 식별자를 검색하는 과정을 환경 레코드(Environment Record) 관점에서 설명하세요.

```jsx
function foo() {
  var x = 1;
  const y = 2;

  function bar() {
    console.log(x + y + z);
  }

  bar();
}

foo();
const z = 3;
```

## Q4. 실행 결과를 예측하고, **렉시컬 스코프**와 **Outer Lexical Environment Reference** 관점에서 `inner` 함수가 `x`를 참조하는 과정을 설명하세요.

## Q5. 빈칸에 들어갈 개념은?

전역 코드 평가 시점에 `var` 변수와 함수 선언문은 전역 환경 레코드의 ( **ㄱ** )에 연결된 `BindingObject`를 통해 전역 객체(`window`)의 프로퍼티와 메서드가 된다. 이 과정 덕분에 선언문 이전에 변수를 참조하거나 함수를 호출하는 호이스팅이 가능해진다.

## Q6. 식별자 결정 과정 단계별로 서술하기

아래 코드에서 `inner` 함수 내부의 `console.log(a + b)`가 실행될 때, 자바스크립트 엔진이 **스코프 체인**을 통해 식별자 `a`를 찾아가는 과정을 단계별로 서술하세요.

```jsx
const a = 100;

function outer() {
  const b = 20;
  function inner() {
    console.log(a + b);
  }
  inner();
}

outer();
```
