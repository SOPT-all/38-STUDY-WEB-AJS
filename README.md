## 👍 스터디 방식

- 매 주차별 선정된 발표자(1~2명)는 문제 풀이를 진행하거나 해당 주차 주제와 관련된 심화 내용 미미나를 진행
- 모든 스터디원은 **아티클(공부 기록)을 작성하고 PR 또는 링크**로 제출 (일요일 23:59까지)
  - 스터디 전까지 최소 2명 이상의 아티클에 배운 점, 질문, 피드백 등 댓글 작성 (스터디 전까지)
- 발표자는 스터디 전날까지 PR로 문제를 출제하고, 모든 스터디원은 스터디 전까지 reivew로 풀이를 기록


<br />

## 📝 문제 제출 가이드

### 🌳 브랜치 규칙

아래와 같은 형식으로 브랜치를 생성하고, `main` 브랜치로 PR(Pull Request)을 보내주세요.

- 브랜치명: `주차/이름`  
  예시:  
  - `week1/soy` 
   
### 📁 폴더 및 파일 
파일명 및 폴더 구조는 자유지만 꼭 본인 디렉토리에서 작업 부탁드립니다! 
퀴즈/내용정리 완료 후, 주차 별 브랜치(`weekn/이름`)에 commit 및 push하시면 됩니다.

#### 📌 폴더 구조
- **`docs/`**  
  - 각 스터디 멤버가 한 주간 공부한 내용이나 관련 아티클을 `.md` 파일 형식으로 업로드하는 폴더입니다.
  - 노션, 블로그 등의 링크 작성도 가능합니다.(md 파일에 작성)

- **`quiz/`**  
  각 주차 별 퀴즈를 출제하는 곳입니다. 

✅ 파일은 **본인 디렉토리** 아래에 정리 (예: `docs/soy/week1/README.md 또는 quiz/soy/week1/quiz.md)

<br />

### 📝 Pull Request 작성 시
- 제목 예시: `[Week1/박소이] 변수, 함수, 스코프 퀴즈 (4장, 12-15장)`
- 문제를 (최소) 4~5개 출제해주세요.
- PR 템플릿에 quiz 복붙해서 올리시면 됩니다.

+) main 브랜치로의 merge는 스터디가 끝난 뒤 일괄적으로 적용하겠습니다!
